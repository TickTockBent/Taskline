The main scheduler that manages and executes tasks.

Scheduler maintains a collection of tasks and executes them according to their cron schedules. It runs in the background and provides methods for managing tasks and controlling the scheduler lifecycle.

§Examples

§Basic Usage

use cronline::{Scheduler, Task};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let scheduler = Scheduler::new();

    // Add tasks
    scheduler.add("0 * * * *", Task::new(|| async {
        println!("Hourly task");
        Ok(())
    })).await?;

    // Start scheduler
    scheduler.start().await?;

    // ... do other work ...

    // Stop scheduler
    scheduler.stop().await?;
    Ok(())
}

§Custom Configuration

use cronline::{Scheduler, SchedulerConfig};
use std::time::Duration;

let config = SchedulerConfig {
    check_interval: Duration::from_millis(100),
    continue_on_error: true,
    shutdown_grace_period: Duration::from_secs(5),
};

let scheduler = Scheduler::with_config(config);

pub async fn add(&self, cron_expr: &str, task: Task) -> Result<String>

Adds a task to the scheduler with a cron schedule.

The task will be executed according to the provided cron expression. Returns a unique task ID that can be used to reference the task later.

§Arguments
  • cron_expr - A cron expression string (e.g., “0 * * * *”)
  • task - The task to schedule
§Returns

Returns the unique ID of the added task, or Err if the cron expression is invalid.

§Examples
use cronline::{Scheduler, Task};

let scheduler = Scheduler::new();

let task_id = scheduler.add("0 * * * *", Task::new(|| async {
    println!("Running hourly");
    Ok(())
})).await?;

println!("Added task with ID: {}", task_id);
§Errors

Returns CronlineError::CronParseError if the cron expression is invalid.

pub async fn start(&self) -> Result<()>

Starts the scheduler in the background.

The scheduler begins monitoring tasks and executing them according to their schedules. This method returns immediately, allowing you to continue other work while the scheduler runs.

§Returns

Returns Ok(()) if the scheduler started successfully, or Err if it’s already running.

§Examples
let scheduler = Scheduler::new();
// Add tasks...
scheduler.start().await?;
// Do other work...
scheduler.stop().await?;
§Errors

Returns CronlineError::SchedulerError if the scheduler is already running.

pub async fn stop(&self) -> Result<()>

Stops the scheduler gracefully.

The scheduler will stop accepting new task executions and wait for running tasks to complete (up to the configured shutdown_grace_period).

§Returns

Returns Ok(()) if the scheduler stopped successfully, or Err if it’s not running.

§Examples
let scheduler = Scheduler::new();
scheduler.start().await?;
// ... later ...
scheduler.stop().await?;
§Errors

Returns CronlineError::SchedulerError if the scheduler is not running.
