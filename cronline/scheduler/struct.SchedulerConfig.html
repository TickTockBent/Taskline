Expand description

Configuration options for the scheduler.

SchedulerConfig controls the schedulerโs behavior, including how often it checks for due tasks, error handling strategy, and shutdown behavior.

ยงExamples

use cronline::SchedulerConfig;
use std::time::Duration;

let config = SchedulerConfig {
    check_interval: Duration::from_millis(100),  // Check every 100ms
    continue_on_error: true,                     // Keep running on errors
    shutdown_grace_period: Duration::from_secs(10), // 10s shutdown grace period
};

How frequently to check for tasks that need to be executed.

Lower values provide more precise scheduling but increase CPU usage. Default: 500ms

ยงcontinue_on_error: bool

Whether to continue running when a task fails.

If true, task failures donโt stop the scheduler. If false, the scheduler stops when any task fails. Default: true

ยงshutdown_grace_period: Duration

How long to wait for tasks to complete during shutdown.

When stopping the scheduler, this is the maximum time to wait for running tasks to finish. Default: 30s
